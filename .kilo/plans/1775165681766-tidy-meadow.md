# Plan: Fix ESP — restore working logic from old version

## Root Cause Analysis

Сравнение текущей версии (`src/`) со старой рабочей (`cs2-external-main/src/`) выявило **критическое отличие** в `game_manager.cpp`:

### Проблема 1: `tooFar` distance check logic (строки 251-259)

**OLD (working):**
```cpp
if (tooFar && !s_readBones.load() && !s_readWeapons.load()) {
    s_invalidSlotCache[slot] = INVALID_SLOT_SKIP_FRAMES;
    continue;
}
```

**CURRENT (broken):**
```cpp
if (tooFar) {
    // empty block — dead code
}
```

В старой версии далёкие сущности пропускались когда кости/оружие отключены. В текущей версии пустой блок не делает ничего, но последующие проверки `if (!tooFar && ...)` корректно пропускают чтение костей/оружия. Однако пустой блок — мёртвый код, а логика обработки далёких сущностей изменилась.

### Проблема 2: `INVALID_SLOT_SKIP_FRAMES` changed from 60 → 15

**OLD:** `static constexpr int INVALID_SLOT_SKIP_FRAMES = 60;`
**CURRENT:** `Constants::INVALID_SLOT_SKIP_FRAMES = 15;`

Сущности с ошибками валидации повторяются через 15 кадров вместо 60, что вызывает мерцание ESP.

### Проблема 3: `s_prevPositions.clear()` on map change

**OLD:** Не очищает `s_prevPositions` при смене карты
**CURRENT:** Очищает `s_prevPositions` при смене карты

Это может вызвать кратковременный сбой позиций после смены карты.
