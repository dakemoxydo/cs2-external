# CS2 Overlay — Project Analysis & Improvement Plan

## Текущее состояние проекта

Проект представляет собой **external overlay-чит для CS2** на C++20 с использованием ImGui + DirectX 11. Архитектура многопоточная (Memory Thread + Render Thread), с системой конфигов на основе JSON-реестра и модульной системой фич через интерфейс `IFeature`.

### Сильные стороны
- Чёткое разделение на слои (Core, Features, Render, Config, Input)
- Многопоточная архитектура с разделением чтения памяти и рендера
- Модульная система фич с `IFeature` интерфейсом
- Автоматическая сериализация конфигов через Config Registry
- Система тем оформления меню (7 тем)
- External-подход (без инъекции DLL)

### Ключевые проблемы
- **Критические race condition** в статических переменных без синхронизации
- **Гарантированный deadlock** в логгере
- **Утечки памяти** в UI-компонентах ImGui
- **Отсутствие/null pointer проверки** в рендерере
- **InputManager — полностью заглушка** (все методы пустые)
- **Дублирование кода** во множестве мест
- **Небезопасная работа с памятью** через `reinterpret_cast`

## Приоритет 1: Критический (стабильность, безопасность, утечки ресурсов)

### 1.1. Eliminate Race Conditions on Static Mutable State
**Затронутые файлы:**
- `src/core/process/process.cpp:28-30` — `hProcess`, `processId`, `s_ntRvm`
- `src/core/process/stealth.cpp:22-23, 144-145` — `s_ntSip`, `s_randGen`, `s_sleepDist`
- `src/core/sdk/game_manager.h:52` — `s_interpolationFactor`
- `src/config/config_manager.cpp:31` — `LastError`
- `src/utils/timer.h:12-13` — `deltaTime`, `currentFPS`

**Проблема:** Статические переменные читаются/записываются из разных потоков без mutex/atomic. `std::mt19937_64` не thread-safe — конкурентные вызовы портят внутреннее состояние.

**Решение:**
- Заменить скалярные static-переменные на `std::atomic<T>` где возможно
- Для сложных структур использовать `std::shared_mutex` (read-heavy) или `std::mutex`
- Для RNG — thread_local экземпляр или mutex

### 1.2. Fix Logger Deadlock
**Затронутые файлы:** `src/utils/logger.cpp:13-19`

**Проблема:** `Init()` захватывает `std::mutex`, затем вызывает `Info()` → `Write()`, которая пытается захватить тот же не-рекурсивный mutex — **гарантированный deadlock**.

**Решение:**
- Использовать `std::recursive_mutex` вместо `std::mutex`
- Либо вынести `Info()` за пределы блокировки
- Либо сделать `Write()` internal-метод без блокировки, а публичные методы — с блокировкой

### 1.3. Fix Memory Leak in ImGui StateStorage
**Затронутые файлы:** `src/render/menu/ui_components.cpp:110-114`

**Проблема:** `ImGui::MemAlloc()` для анимации тоггла никогда не освобождается. ImGui StateStorage не освобождает raw-указатели автоматически.

**Решение:** Использовать `ImGui::GetStateStorage()->GetFloat/SetFloat()` вместо raw-указателей, или хранить анимацию как `int` (bitcast float→int).

### 1.4. Add Null Pointer Validation in Renderer
**Затронутые файлы:** `src/render/renderer/renderer.cpp:41-43, 67-74`

**Проблема:** `BeginFrame()` и `EndFrame()` разыменовывают `pContext`, `pRenderTargetView`, `pSwapChain` без проверок. `CreateRenderTargetView` не проверяет HRESULT.

**Решение:**
- Добавить null-чеки перед разыменованием в `BeginFrame/EndFrame`
- Проверять HRESULT от `CreateRenderTargetView`
- В `Init()` возвращать false при любой ошибке инициализации

### 1.5. Fix InputManager Stub Implementation
**Затронутые файлы:** `src/input/input_manager.cpp:7-11`

**Проблема:** `Poll()` пустой, `IsKeyDown/IsKeyPressed` всегда возвращают `false`. Любой код, использующий эти методы, молча не работает.

**Решение:** Реализовать полноценный ввод через `GetAsyncKeyState` или буферизированную систему состояний клавиш.

### 1.6. Fix Stack Buffer Overflow in FetchHTTP
**Затронутые файлы:** `src/core/sdk/offset_loader.cpp:34-39`

**Проблема:** `buf[n] = '\0'` может записать за пределы `char buf[8192]`, если `InternetReadFile` вернёт ровно 8192 байта.

**Решение:** Использовать `char buf[8193]` или читать максимум `sizeof(buf) - 1` байт.

### 1.7. Fix Header Guard Order Bug
**Затронутые файлы:** `src/features/bomb/bomb.h:1-2`, `src/features/debug_overlay/debug_overlay.h:1-2`

**Проблема:** `#include` стоит ДО `#pragma once`, что приводит к повторной обработке инклюдов.

**Решение:** Переместить `#pragma once` на первую строку.

### 1.8. Fix PEB Write Without VirtualProtect
**Затронутые файлы:** `src/core/process/stealth.cpp:87-88`

**Проблема:** Прямая запись в `RTL_USER_PROCESS_PARAMETERS` без `VirtualProtect` может вызвать access violation на некоторых конфигурациях Windows.

**Решение:** Добавить `VirtualProtect` с `PAGE_READWRITE` перед записью и восстановить после.

### 1.9. Add Compiler Warning Flags
**Затронутые файлы:** `CMakeLists.txt`

**Проблема:** Отсутствуют флаги предупреждений компилятора (`/W4` для MSVC). `_CRT_SECURE_NO_WARNINGS` маскирует потенциальные уязвимости.

**Решение:**
- Добавить `target_compile_options(cs2overlay PRIVATE /W4 /WX)` 
- Убрать `_CRT_SECURE_NO_WARNINGS`, заменить unsafe-функции на `_s` варианты

## Приоритет 2: Важный (эффективность, масштабируемость, UX)

### 2.1. Eliminate Code Duplication — RCS Logic
**Затронутые файлы:**
- `src/features/aimbot/aimbot.cpp:130-131, 169-170`
- `src/features/rcs/rcs.cpp:80-81`

**Проблема:** Формула компенсации отдачи дублируется в aimbot и rcs.

**Решение:** Вынести в `Core::Math::ApplyRCSCompensation(pitch, yaw, punchDelta)`.

### 2.2. Return Players by const Reference, Not by Value
**Затронутые файлы:**
- `src/core/game/game_manager_getters.cpp:15-19`
- `src/features/aimbot/aimbot.cpp:96`
- `src/features/esp/esp.cpp:50`
- `src/features/triggerbot/triggerbot.cpp:56`
- `src/features/debug_overlay/debug_overlay.cpp:64`

**Проблема:** `GetRenderPlayers()` копирует весь `std::vector<Entity>` при каждом вызове. Вызывается 4+ раз за кадр.

**Решение:** Вернуть `const std::vector<SDK::Entity>&` или использовать `std::shared_ptr<const std::vector<...>>` для lock-free чтения.

### 2.3. Fix `ApplyMenuTheme` Called Every Frame
**Затронутые файлы:** `src/render/menu/menu.cpp:30`

**Проблема:** Тема применяется каждый кадр в `Menu::Render()`, что wasteful и может мешать анимациям ImGui.

**Решение:** Применять тему только при изменении — кэшировать `lastAppliedTheme` и сравнивать.

### 2.4. Fix `shouldClose` Flag Never Set
**Затронутые файлы:** `src/render/menu/menu.cpp:19`, `src/render/menu/menu.h:13`

**Проблема:** `shouldClose` объявлен, но никогда не устанавливается в `true`. Мёртвый код, указывающий на незавершённую реализацию.

**Решение:** Либо реализовать установку флага (например, при закрытии меню), либо удалить.

### 2.5. Fix `LoadDefault` No-Op
**Затронутые файлы:** `src/config/config_manager.cpp:248-250`

**Проблема:** Функция ничего не делает. Сброс конфигов к дефолтным значениям невозможен.

**Решение:** Реализовать — пересоздать `GlobalSettings` с дефолтными значениями и сохранить.

### 2.6. Fix `InputManager::KeyLabel` Static Buffer Race
**Затронутые файлы:** `src/render/menu/ui_components.cpp:23-26`

**Проблема:** Статический буфер `char buf[8]` — конкурентные вызовы перезаписывают друг друга.

**Решение:** Вернуть `std::string` по значению или использовать thread_local буфер.

### 2.7. Fix `SettingHotkey` Single-Key Limitation
**Затронутые файлы:** `src/render/menu/ui_components.cpp:183`

**Проблема:** `static int *s_listening = nullptr` — только один хоткей может быть в режиме "слушания".

**Решение:** Использовать map `ImGuiID → int*` для поддержки нескольких одновременных хоткеев.

### 2.8. Optimize `BuildRegistry()` — Cache Result
**Затронутые файлы:** `src/config/config_manager.cpp:50-132`

**Проблема:** Реестр из ~50 записей перестраивается при каждом `Save()` и `Load()`.

**Решение:** Сделать `static const` кэш или инициализировать один раз при старте.

### 2.9. Fix `GetRenderPlayers()` Vector Copy Under Lock
**Затронутые файлы:** `src/core/game/game_manager_getters.cpp:15-19`, `src/core/game/game_manager.h:76`

**Проблема:** Копирование вектора происходит под `std::mutex`, блокируя memory thread.

**Решение:** Использовать `std::shared_mutex` + `std::shared_lock` для чтения, или double-buffering с `std::shared_ptr`.

### 2.10. Fix `s_nameCache` Unbounded Growth
**Затронутые файлы:** `src/core/game/game_manager.cpp:58, 304-307`

**Проблема:** Кэш имён никогда не эвиктится — растёт неограниченно.

**Решение:** Добавить LRU-эвикцию или ограничение по размеру (например, 64 записи).

### 2.11. Fix `WndProc` Return Type Mismatch
**Затронутые файлы:** `src/render/overlay/overlay.cpp:216`

**Проблема:** Возвращает `true` (bool) вместо `LRESULT`. Компилируется из-за неявного преобразования, но возвращает `1` вместо `0`.

**Решение:** Заменить `return true` на `return 0` или `return 1` явно.

### 2.12. Fix `GetWindowRect` No Error Check
**Затронутые файлы:** `src/render/overlay/overlay.cpp:119`

**Проблема:** Если `GetWindowRect` упадёт (окно CS2 закрылось), оверлей создастся с мусорными координатами.

**Решение:** Проверить возвращаемое значение, при ошибке — повторный поиск окна или возврат false.

### 2.13. Replace Deprecated `SetProcessDPIAware`
**Затронутые файлы:** `src/render/overlay/overlay.cpp:84`

**Проблема:** Функция deprecated с Windows 8.1.

**Решение:** Заменить на `SetProcessDpiAwarenessContext(DPI_AWARENESS_CONTEXT_PER_MONITOR_AWARE_V2)`.

### 2.14. Fix Hardcoded 60Hz Refresh Rate
**Затронутые файлы:** `src/render/renderer/renderer.cpp:17-18`

**Проблема:** Refresh rate захардкожен на 60Hz. На 144/240Hz мониторах могут быть проблемы.

**Решение:** Определить текущий refresh rate через `EnumDisplaySettings` или использовать `DXGI_SWAP_EFFECT_FLIP_DISCARD` с `RefreshRate = {0, 1}` (auto-detect).

### 2.15. Fix Feature Coupling (Aimbot ↔ RCS)
**Затронутые файлы:**
- `src/features/aimbot/aimbot.cpp:93-94`
- `src/features/rcs/rcs.cpp:69-70`

**Проблема:** Aimbot вызывает `RCSSystem::IsWeaponSupported()`, RCS читает `Config::Settings.aimbot.hotkey`. Bidirectional coupling.

**Решение:** Создать общий интерфейс `IRecoilCompensation` и инжектить через FeatureManager.

### 2.16. Fix `FeatureManager::features` Public Static with No Thread Safety
**Затронутые файлы:** `src/features/feature_manager.h:18`, `src/features/feature_manager.cpp:43-55`

**Проблема:** Публичный static-вектор без инкапсуляции и синхронизации.

**Решение:** Сделать private, добавить mutex-protected методы `Register()`, `Unregister()`, `ForEach()`.

### 2.17. Fix `ApplySettings` String-Based Feature Matching
**Затронутые файлы:** `src/config/config_manager.cpp:216-234`

**Проблема:** O(n*m) сравнения строк, fragile при переименовании фич.

**Решение:** Использовать enum FeatureID или registry-based маппинг.

### 2.18. Fix `string_utils.h` UB with Non-ASCII
**Затронутые файлы:** `src/utils/string_utils.h:7-8, 12-13`

**Проблема:** `::tolower(char)` с отрицательным char — undefined behavior.

**Решение:** `[](unsigned char c) { return ::tolower(c); }`

### 2.19. Fix `ConfigManager::LastError` Data Race
**Затронутые файлы:** `src/config/config_manager.cpp:31`

**Проблема:** `LastError` записывается из `Save()` и `Load()` без синхронизации.

**Решение:** Добавить mutex или сделать `std::atomic<std::string>` (через кастомный wrapper).

### 2.20. Fix `FeatureManager::features` Iteration Without Lock
**Затронутые файлы:** `src/config/config_manager.cpp:216-234`

**Проблема:** Итерация по `FeatureManager::features` без синхронизации — data race при регистрации/удалении фич.

**Решение:** Добавить мьютекс в FeatureManager, использовать lock при итерации.

## Приоритет 3: Оптимизирующий (UX, технический долг, инновации)

### 3.1. Optimize ESP Bone Rendering
**Затронутые файлы:** `src/features/esp/esp.cpp:222-227`

**Проблема:** `WorldToScreen` вызывается дважды для каждой кости в каждом соединении (~30 вызовов на игрока).

**Решение:** Преобразовать все кости в screen space один раз, затем рисовать линии.

### 3.2. Optimize `std::to_string` Allocations in ESP Render Loop
**Затронутые файлы:** `src/features/esp/esp.cpp:158, 195-196`

**Проблема:** `std::to_string` делает heap-аллокации каждый кадр для каждого видимого игрока.

**Решение:** Использовать `snprintf` с stack-буфером `char buf[16]`.

### 3.3. Optimize `ClampAngle` — Replace Loops with fmod
**Затронутые файлы:** `src/core/math/math.cpp:48-54`

**Проблема:** `while`-циклы для нормализации углов могут итерировать много раз.

**Решение:** `return fmodf(a + 180.0f, 360.0f) - 180.0f;`

### 3.4. Eliminate `DrawList` Color Conversion Duplication
**Затронутые файлы:** `src/render/draw/draw_list.cpp:6-68`

**Проблема:** `ImGui::ColorConvertFloat4ToU32` повторяется 6 раз в 70-строчном файле.

**Решение:** Private helper `static ImU32 ToImU32(const float color[4])`.

### 3.5. Make `DrawList` Methods Static
**Затронутые файлы:** `src/render/draw/draw_list.h:6-16`

**Проблема:** Все методы работают с `ImGui::GetBackgroundDrawList()` и не имеют state.

**Решение:** Сделать методы `static`, убрать необходимость инстанцирования.

### 3.6. Optimize `ApplyMenuTheme` — Use Data-Driven Approach
**Затронутые файлы:** `src/render/menu/menu_theme.cpp:1-245`

**Проблема:** 242 строки императивных `if/else if` для 7 тем.

**Решение:** Определить темы как compile-time массивы структур `ThemeData`, применять через индекс.

### 3.7. Fix Duplicate Midnight Theme in ImGuiManager
**Затронутые файлы:** `src/render/renderer/imgui_manager.cpp:27-79`, `src/render/menu/menu_theme.cpp:6-242`

**Проблема:** Тема Midnight — точная копия стиля из `ImGuiManager::Init()`.

**Решение:** Вынести общий стиль в отдельную функцию, использовать в обоих местах.

### 3.8. Optimize `FetchHTTP` — Cache User-Agent, URL Base
**Затронутые файлы:** `src/core/sdk/offset_loader.cpp:14, 154-157, 203-206`

**Проблема:** User-agent и базовый URL конкатенируются каждый вызов.

**Решение:** `static constexpr` строки.

### 3.9. Optimize `FindFieldInClientJson` — Single Pass
**Затронутые файлы:** `src/core/sdk/offset_loader.cpp:87-102`

**Проблема:** Для каждого из 17 оффсетов — линейный проход по всему JSON. O(17*N).

**Решение:** Один проход с построением lookup map, затем O(1) для каждого поля.

### 3.10. Fix `s_pawnListCache` Cleared Every Frame
**Затронутые файлы:** `src/core/game/game_manager.cpp:166`

**Проблема:** `memset` кэша каждый кадр уничтожает его пользу.

**Решение:** Кэшировать между кадрами, инвалидировать только при изменении entity list.

### 3.11. Optimize `players.reserve(16)` → `reserve(MAX_PLAYERS)`
**Затронутые файлы:** `src/core/game/game_manager.cpp:82`

**Проблема:** Reserve меньше MAX_PLAYERS可能导致 реаллокации.

**Решение:** `players.reserve(MAX_PLAYERS)` или разумный upper bound (20).

### 3.12. Fix `GetAsyncKeyState` Called Multiple Times Per Frame
**Затронутые файлы:**
- `src/features/rcs/rcs.cpp:69-70`
- `src/features/aimbot/aimbot.cpp:81`
- `src/main.cpp:149`

**Проблема:** Один и тот же ключ опрашивается несколько раз за кадр.

**Решение:** Кэшировать состояние клавиш в InputManager один раз за кадр.

### 3.13. Remove Dead Code
**Затронутые файлы:**
- `src/core/sdk/updater.cpp` — deprecated, только include
- `src/core/sdk/entity.cpp` — пустой
- `src/core/sdk/player.h` — класс без реализации
- `src/core/process/stealth.cpp:147-153` — `RandomizeTiming` не вызывается
- `src/core/process/stealth.cpp:22, 30-32` — `s_ntSip` разрешается но не используется

**Решение:** Удалить все мёртвые файлы/функции.

### 3.14. Fix Mixed Russian/English Comments
**Затронутые файлы:** Множественные

**Проблема:** Комментарии на русском и английском, что затрудняет поддержку.

**Решение:** Стандартизировать на один язык (рекомендуется английский для совместимости).

### 3.15. Optimize Logger Flush Frequency
**Затронутые файлы:** `src/utils/logger.cpp:98`

**Проблема:** `flush()` на каждую строку лога — severe I/O bottleneck.

**Решение:** Буферизированные записи, flush только на Warn/Error или по таймеру.

### 3.16. Fix `build.bat` — Use `vswhere.exe` for VS Discovery
**Затронутые файлы:** `build.bat:23-54`

**Проблема:** Захардкоженные пути VS 2019/2022.

**Решение:** Использовать `vswhere.exe` для динамического обнаружения.

### 3.17. Fix PE Mutation Binary Bloat
**Затронутые файлы:** `scripts/mutate_signature.ps1:56-61`

**Проблема:** При повторных билдах без `.mutate` секции — 256 байт добавляются каждый раз.

**Решение:** Проверять существование секции перед добавлением, или очищать перед билдом.

### 3.18. Standardize Language for Comments and Logs
**Затронутые файлы:** Все файлы проекта

**Проблема:** Смесь русского и английского.

**Решение:** Выбрать один язык и привести все комментарии/логи к нему.

### 3.19. Add `OnEnable`/`OnDisable` Implementations
**Затронутые файлы:** Все feature `.cpp` файлы

**Проблема:** `IFeature` предоставляет хуки, но ни одна фича их не реализует.

**Решение:** Реализовать для сброса состояния (aimbot → clear `s_lastTarget`, RCS → clear `s_oldPunch`).

### 3.20. Fix `world_to_screen.h` Missing Implementation
**Затронутые файлы:** `src/render/draw/world_to_screen.h:1-11`

**Проблема:** Класс объявляет `static bool Convert()` но нет `.cpp` файла.

**Решение:** Либо добавить реализацию, либо удалить если не используется.
