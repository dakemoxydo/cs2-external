# Plan: UI/UX Improvements for Visuals Tab and Menu Design

## Analysis of Current State

**Current UI Architecture:**
- 900x650 window, 48px custom header, 180px sidebar, 2-column card layout
- 4 tabs: LEGIT, VISUALS, MISC, SETTINGS
- Custom widgets: `SettingToggle` (animated), `SettingColor`, `SettingHotkey`
- 7 theme presets with full color customization
- Cards via `UI::BeginCard()`/`EndCard()` with accent stripe titles

**Current Problems:**
1. **ESP card is a flat wall of toggles** — 15+ settings with no visual grouping
2. **Visuals tab layout is unbalanced** — ESP card is very long, Footsteps ESP is medium, Radar+World Options are short
3. **Color pickers are misaligned** — pushed to far right via `SameLine(GetContentRegionAvail().x - ...)`, creating awkward gaps
4. **No visual feedback** on which features are enabled — card titles don't show status
5. **Footsteps ESP settings are hardcoded in tab_visuals.cpp** instead of using the feature's RenderUI pattern consistently
6. **No quick access** to most-used toggles — user must scroll through all settings
7. **Cards have no "enable all/disable all"** for grouped settings
8. **No visual preview** of what ESP elements will look like
9. **Sidebar tabs are plain text** — no icons or active indicators beyond button color
10. **World Options card is underutilized** — only has bomb timer toggle

## Phase 2: Improved UI Components

### File: `src/render/menu/ui_components.h` + `ui_components.cpp`

**New component: `SettingToggleWithColor`**
- Combines toggle + color picker on same line with proper spacing
- Label on left, toggle in middle, color picker on right
- Eliminates the awkward `SameLine(GetContentRegionAvail().x - ...)` pattern

**New component: `SectionHeader`**
- Styled collapsible section header with accent color
- Shows enabled/disabled indicator dot
- Smooth expand/collapse animation

**New component: `FeatureStatusDot`**
- Small colored dot (green=enabled, red=disabled) for card titles
- Added to `BeginCard()` — shows feature status at a glance

**Improved: `BeginCard`**
- Add optional status indicator parameter
- Add subtle hover effect on card background
- Add subtle gradient to card title area

## Design Principles

1. **Progressive disclosure** — Advanced settings hidden in collapsible sections
2. **Visual hierarchy** — Enable toggles are larger/more prominent than sub-settings
3. **Consistent spacing** — Uniform padding, margins, and element sizing
4. **Status at a glance** — Color dots/icons show feature state immediately
5. **Grouped logically** — Related settings are visually grouped together
6. **Theme-aware** — All new components respect the active theme colors
