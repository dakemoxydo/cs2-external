### 1.2 Add Sound ESP Feature
**File:** `src/features/sound_esp/sound_esp.h` (new)
**File:** `src/features/sound_esp/sound_esp.cpp` (new)

**How it works:**
- CS2 has a sound system that tracks recent sound events (footsteps, weapon fire, reloads, etc.)
- Each sound event has: position, owner (entity handle), volume, type
- Read sound events from memory via `dwSoundSystem` offset from cs2-dumper
- Store recent sounds in a time-based cache, render them as indicators on screen

**Implementation:**
- `SoundEsp::Update()` — reads sound events from memory each frame
- Maintains a `std::vector<SoundEvent>` cache with timestamps
- Expired sounds (> `duration` seconds) are removed
- `SoundEsp::Render()` — converts sound world positions to screen and draws indicators

**Sound Event structure:**
```cpp
struct SoundEvent {
    SDK::Vector3 position;
    float timestamp;
    float volume;
    int ownerHandle;
    bool isFootstep;
};
```

## Part 3: Lazy Loading Features

### 3.1 Add Lazy Init to IFeature
**File:** `src/features/feature_base.h` — add to `IFeature`:
```cpp
virtual void Initialize() {}  // Called once on first enable
bool IsInitialized() const { return initialized; }

    bool initialized = false;
```

Modify `SetEnabled()`:
```cpp
void SetEnabled(bool state) {
    if (enabled != state) {
        if (state && !initialized) {
            Initialize();
            initialized = true;
        }
        enabled = state;
        if (enabled) OnEnable();
        else OnDisable();
    }
}
```

### 3.2 Change RegisterAll to NOT create all features upfront
**File:** `src/features/feature_manager.h` — change:
```cpp
// Instead of registering all at once, use factory pattern
using FeatureFactory = std::function<std::unique_ptr<IFeature>()>;
static void RegisterFeature(std::string_view name, FeatureFactory factory);
static void RegisterAll(); // calls RegisterFeature for each
static void EnsureFeatureInitialized(std::string_view name);
```

**File:** `src/features/feature_manager.cpp` — change:
```cpp
struct FeatureSlot {
    std::string name;
    FeatureFactory factory;
    std::unique_ptr<IFeature> instance;
};
static std::vector<FeatureSlot> featureSlots;

void RegisterFeature(std::string_view name, FeatureFactory factory) {
    featureSlots.push_back({std::string(name), factory, nullptr});
}

void UpdateAll() {
    for (auto& slot : featureSlots) {
        if (!slot.instance) continue; // not yet created
        if (slot.instance->IsEnabled())
            slot.instance->Update();
    }
}

void RenderAll(Render::DrawList& drawList) {
    for (auto& slot : featureSlots) {
        if (!slot.instance) continue;
        if (slot.instance->IsEnabled())
            slot.instance->Render(drawList);
    }
}

// Called by ApplySettings when a feature's enabled state changes
void EnsureFeatureInitialized(std::string_view name) {
    for (auto& slot : featureSlots) {
        if (slot.name == name && !slot.instance) {
            slot.instance = slot.factory();
            slot.instance->SetEnabled(true); // triggers Initialize()
            return;
        }
    }
}
```

### 3.3 Modify ApplySettings to lazy-init
**File:** `src/config/config_manager.cpp` — in `ApplySettings()`:
```cpp
void ConfigManager::ApplySettings() {
    // Check each feature's enabled state and lazy-init if needed
    if (Settings.esp.enabled)
        FeatureManager::EnsureFeatureInitialized("ESP");
    if (Settings.aimbot.enabled)
        FeatureManager::EnsureFeatureInitialized("Aimbot");
    // ... same for all features
}
```
